# Codebase audit — 30 avril 2026

Sweep des 23 PRs lettrées (A → W, PRs #69 → #90) + recherche de code orphelin. Objectif : identifier les promesses dormantes, les fix qui patchent un bug sans le résoudre, et le code mort qui pollue le mental load.

## Méthode

Pour chaque PR : lecture du commit, recherche des fichiers/endpoints livrés, vérification des call sites côté client et instrumentation. Verdict en 4 catégories :

- **alive** — feature utilisée en prod, instrumentée, observable
- **dormant** — code shippé mais jamais activé ou jamais appelé
- **broken** — fix empilé sur un bug non résolu, ou implémentation partielle
- **kill** — orphelin pur, à supprimer

## Findings critiques

### 1. PR I (A/B testing) est totalement dormant

`src/app/lib/experiments.ts` (168 lignes) définit :
- 2 expérimentations registry : `hero_cta_copy`, `surprise_loading_emoji`
- Hashing FNV-1a stable pour le bucketing
- Hook `useExperiment()` React
- Endpoint admin `/admin/experiments/results`
- Auto-attachement des assignments à chaque event analytics (champ `_exp`)

**Mais** : `useExperiment()` n'est appelé nulle part dans le code (uniquement référencé en JSDoc). Le composant Hero utilise `t("hero.cta")` hardcodé au lieu d'une variante. Aucun branchement réel.

**Coût d'inaction** : 168 LOC + 2KB bundle + mental load à chaque relecture.

**Recommandation** : **kill**. Si on veut faire de l'A/B plus tard, on ramènera du code mieux ciblé sur le test précis qu'on veut lancer. Le code "infrastructure pour le futur" sans utilisateur premier est un classique anti-pattern.

### 2. PR M (weekly digest) est broken

`sendPackReadyEmail` et `sendLowCreditsEmail` (lignes 926+) sont câblés correctement (déclenchés depuis `/analyze/surprise-me` et la déduction de crédits). Mais `sendWeeklyDigestEmail` (ligne 967) est défini et **jamais appelé** — pas de cron, pas de webhook, rien.

**Recommandation** : **kill** la fonction `sendWeeklyDigestEmail` + son template. Si on veut un digest plus tard, on l'écrira en sachant déjà qui le déclenche. Garder du code mort "au cas où" est le chemin vers le 12k-ligne monolithe qu'on cherche déjà à découper.

### 3. PR T (style step) — dormance conditionnelle

L'étape de sélection de style existe dans `SurprisePage.tsx` (`idleView === "style"`) mais PR U la désactive par la gate `styleCatalogReady`. Tant que personne n'a cliqué "Seed full catalog", l'étape est invisible.

**Aujourd'hui** : PR V a unblocké le seed (504 fixé), PR W a corrigé la directive. **Dès que tu cliques "Seed", T s'active.**

**Recommandation** : ne pas killer. La dormance est conditionnelle et résoluble en 5 min côté admin.

# ORA Studio

**Your Brand. Amplified.**

ORA Studio is a multi-modal AI aggregator (text, image, video, audio) that transforms a master message into brand-compliant content across every format -- email, LinkedIn, ads, landing pages, stories, newsletters.

15 AI agents. One brand voice. One sentence to full multi-channel campaign in 15 minutes.

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React 18 + TypeScript + Vite 6 |
| Styling | Tailwind CSS v4 + shadcn/ui + Radix UI |
| Routing | React Router 7 (Data mode) |
| Animation | Motion (ex Framer Motion) |
| Icons | Lucide React |
| Backend | Supabase Edge Functions (Hono v4.4.2) |
| Database | Supabase PostgreSQL (KV store pattern) |
| Auth | Supabase Auth (email/password + Google OAuth) |
| Storage | Supabase Storage (private buckets) |

## AI Providers

| Modality | Primary | Secondary | Tertiary |
|----------|---------|-----------|----------|
| Text (LLM) | APIPod (OpenAI, Anthropic, Google models) | -- | -- |
| Image | Runware | FAL AI | Replicate |
| Video | Runware | FAL AI | Replicate |
| Audio | Replicate (MusicGen) | -- | -- |

Fallback chain: if the primary provider fails, the system automatically tries the next one.

## Getting Started

### Prerequisites

- Node.js 18+
- pnpm (recommended) or npm
- A Supabase project with Edge Functions enabled

### Installation

```bash
# Clone the repo
git clone https://github.com/YOUR_USER/YOUR_REPO.git
cd YOUR_REPO

# Install dependencies
pnpm install

# Copy environment template
cp .env.example .env.local
# Then fill in your API keys (see below)

# Start dev server
pnpm dev
```

### Deploy Edge Functions

```bash
# Login to Supabase CLI
supabase login

# Link to your project
supabase link --project-ref YOUR_PROJECT_REF

# Set secrets
supabase secrets set --env-file .env.edge

# Deploy the Edge Function
supabase functions deploy server
```

## Architecture Notes

### Auth Pattern

The auth system uses a **dual-header pattern** to avoid a known `sb.auth.getUser()` hanging issue:
- `Authorization: Bearer <publicAnonKey>` -- for the Supabase gateway
- `X-User-Token: <userJWT>` -- for server-side auth

The server decodes JWTs locally (`decodeJwtPayload`) instead of calling `sb.auth.getUser()`.

### AI Provider Fallback

All image and video generation uses a cascading fallback:
1. Try primary provider (Runware)
2. If it fails, try secondary (FAL AI)
3. If it fails, try tertiary (Replicate)

Text generation goes exclusively through APIPod, which is an OpenAI-compatible gateway supporting OpenAI, Anthropic, and Google models.

### Cost Tracking

Server-side cost tracking logs every AI generation call with provider, model, modality, cost, and latency. Accessible via the Admin page (`/admin`) with built-in SVG/CSS charts.

### Design System

- **No photos, no emojis, no figurative illustrations**
- Premium, minimal, light-theme
- Apple-like clarity + Stripe-like system design
- Single accent color: `--ora-signal: #3b4fc4` (slate-blue)
- Font: Inter (300, 400, 500, 600)
- Generous whitespace, hairline borders, soft shadows
